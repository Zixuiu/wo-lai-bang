# Checklist Generator

**核心功能**：
1. 从用户需求生成 Checklist + 验收标准
2. 每个任务必须紧跟该任务的验收标准
3. 自动执行实现

---

## ⚠️ 严格格式要求（必须遵守）

### 错误写法 ❌

```markdown
## 任务列表
- [ ] 任务1
- [ ] 任务2
- [ ] 任务3

## 验收标准
- ✅ 标准1
- ✅ 标准2
```

**这是错误的！任务和验收标准分开了！**

---

### 正确写法 ✅

```markdown
## P0 - 必须实现

### 1. 功能名称A

**描述**: 简要说明

**修复方案**:
- [ ] 具体任务1-1
- [ ] 具体任务1-2

**验收标准**：
- ✅ 条件A → 结果A
- ✅ 条件B → 结果B
- ❌ 失败条件 → 失败表现

---

### 2. 功能名称B

**描述**: 简要说明

**修复方案**:
- [ ] 具体任务2-1
- [ ] 具体任务2-2

**验收标准**：
- ✅ 条件 → 结果
- ❌ 失败 → 表现
```

## P0 - 必须实现

### 1. 隐私政策页面

**描述**: 用户首次打开App时必须阅读并同意隐私政策，否则无法注册

**修复方案**:
- [ ] 创建 `pages/privacy-policy/privacy-policy.vue` 页面
- [ ] 在 App.vue 的 onLaunch 中检测是否首次启动
- [ ] 首次启动显示全屏隐私政策弹窗，包含"同意"和"查看详情"按钮
- [ ] 用户点击"同意"后记录状态到 Storage，后续不再显示
- [ ] 用户点击"查看详情"跳转到隐私政策页面

**验收标准**：
- ✅ 用户首次打开App → 立即弹出隐私政策全屏弹窗
- ✅ 用户点击"同意" → 弹窗关闭，可以正常使用App
- ✅ 用户点击"查看详情" → 跳转隐私政策页面，阅读后可返回
- ❌ 用户拒绝隐私政策 → 无法注册/登录，提示必须同意
- ❌ 非首次启动 → 仍弹出隐私政策弹窗

---

### 2. 用户协议页面

**描述**: 注册页面必须显示用户协议勾选框，用户必须同意才能注册

**修复方案**:
- [ ] 创建 `pages/terms-of-service/terms-of-service.vue` 页面
- [ ] 在 register.vue 注册表单底部添加协议勾选框
- [ ] 复选框文字包含《用户协议》和《隐私政策》两个链接
- [ ] 点击链接新窗口打开对应页面
- [ ] 未勾选时注册按钮显示禁用状态

**验收标准**：
- ✅ 注册页面底部有勾选框 → 显示《用户协议》和《隐私政策》
- ✅ 勾选协议 → 注册按钮变为可点击状态（绿色）
- ✅ 未勾选协议 → 注册按钮禁用（灰色），点击无反应
- ❌ 协议内容为空 → 跳转后页面无内容
- ❌ 未勾选协议 → 仍能点击注册按钮

## P1 - 重要功能

### 3. 实名认证入口

**描述**: 提供实名认证功能，用户可以提交身份证信息进行认证

**修复方案**:
- [ ] 创建 `pages/verify/verify.vue` 认证页面
- [ ] 页面包含：姓名输入框、身份证号输入框、身份证正面照上传、身份证反面照上传
- [ ] 调用后端API提交认证申请
- [ ] 在个人中心显示认证状态标签（未认证/认证中/已认证/认证失败）
- [ ] 认证状态变更时发送系统通知

**验收标准**：
- ✅ 用户填写信息上传证件后提交 → 显示"认证中"状态
- ✅ 后台审核通过 → 状态变为"已认证"，显示绿色标签
- ✅ 后台审核拒绝 → 状态变为"认证失败"，显示红色标签和原因
- ❌ 认证状态长时间不更新 → 未发送提醒通知
- ❌ 身份证信息泄露 → 验收不通过

## ⚠️ 关键规则（必须遵守）

1. **每个任务必须紧跟该任务的验收标准**
   - 任务1 → 任务1的验收标准 → 分隔线
   - 任务2 → 任务2的验收标准 → 分隔线
   - 任务3 → 任务3的验收标准 → 分隔线

2. **禁止这样写**：
   - ❌ 先把所有任务列出来，再把所有验收标准列出来
   - ❌ 验收标准单独放一个章节

3. **验收标准格式**：
   - `✅ 条件 → 预期结果`
   - `❌ 失败条件 → 失败表现`

4. **修复方案要具体**：
   - ✅ `[ ] 创建 pages/privacy-policy/privacy-policy.vue 页面`
   - ❌ `[ ] 实现隐私政策页面`

5. **每个任务之间用 `---` 分隔**

6. **文件保存位置**：`docs/checklist-{需求简称}.md`

## 执行流程

### Step 1: 分析需求
1. 理解需求范围和目标
2. 拆分为 P0/P1/P2 优先级
3. 每个 P0 项确定具体修复方案

### Step 2: 生成文件
```javascript
const title = "上架应用市场"
const content = `# 上架应用市场

## P0 - 必须实现

### 1. 隐私政策页面

**描述**: ...

**修复方案**:
- [ ] ...

**验收标准**：
- ✅ ...

---

### 2. 用户协议页面
...`
Write({ content, file_path: `d:\\code空间\\wo-laibang-app\\docs\\checklist-${title}.md` })
```

### Step 3: 创建 TodoWrite
```javascript
TodoWrite({
  todos: [
    { id: "1", content: "隐私政策页面", status: "pending", priority: "high" },
    { id: "2", content: "用户协议页面", status: "pending", priority: "high" },
  ]
})
```

### Step 4: 执行并更新
- 每完成一项：更新文件 `[x]` + TodoWrite 状态
- 全部完成：更新执行记录表格状态为"已完成"
